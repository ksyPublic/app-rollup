네이밍규칙

아이디(ID)

  • 카멜 케이스기법을 사용한다.
  • 지정된 레이아웃을 제외하고 UI스타일 지정을 위하여는 id를 사용하지 않는다.
  • 앵커로서 사용되는 엘리먼트 또는 User Interface 동작을 위하여 DOM 선택자로서의 id 지정은 가능하다.
  • 예약어가 있는 경우 예약어를 사용한다.

클래스(CLASS)

  • 네이밍은 하이픈(-) 사용을 기본으로 한다.
  • 시작의 이름은 영문 소문자를 사용하며 두 번째 단어부터 하이픈으로 연결하며 대문자는 사용하지 않는다.
  • 영문 소문자, 숫자, 하이픈만 사용할 수 있다.
  • BEM 방식을 사용한다.
  • Block Element Modifier 방식을 사용한다.
기본적 클래스 예약어
예약어 설명
.modal-* 레이어형태 팝업
.heading .heading--* 타이틀 요소 지정
.btn-x 버튼 요소 지정
.hide 숨김요소 : 시각적으로만 숨김처리되는 요소로, 스크린리더를 통해서는 정보제공되는 대상
.none 숨김요소 : 시각, 청각요소 모두 숨김처리대상. 주로 UI동작관련 하여 필요시 사용.
개발에서 display 조절필요시 inline style 이 아닌 클래스 추가삭제 컨트롤을 권함.

기타 예약어 [data-*]

기타 예약어
예약어 값 Default 설명
